# Redemption

How minting and redemption work in the Horizon protocol.

---

## The PY Index

The protocol tracks a **PY Index** that determines minting and redemption rates:

<Formula display>{"\\text{Index}_{PY} = \\max(\\text{rate}_{current}, \\text{Index}_{stored})"}</Formula>

### Key Properties

- **Monotonically non-decreasing** — Only goes up or stays the same
- **Captures peak exchange rate** — Protects against temporary drops
- **Shared across all users** — Same index for everyone in a market

<Callout type="info">
The PY Index ensures PT holders are protected even if the underlying asset temporarily loses value.
</Callout>

## Minting (Deposit)

When you deposit SY to mint PT and YT:

<Formula display>{"\\text{Amount}_{minted} = \\text{SY}_{deposited} \\times \\text{Index}_{PY}"}</Formula>

You receive **equal amounts** of PT and YT.

### Example

## Pre-Expiry Redemption

Before maturity, you can redeem by combining **equal amounts** of PT and YT:

<Formula display>{"\\text{SY}_{received} = \\frac{\\text{Tokens}_{burned}}{\\text{Index}_{PY}}"}</Formula>

### Requirements

- Must have **matching PT and YT** amounts
- Same market (token + maturity)
- Returns **SY tokens**

### Example

<Table>
  <TableBody>
    <TableRow>
      <TableCell>PT burned</TableCell>
      <TableCell><Formula>{"105 \\text{ PT}"}</Formula></TableCell>
    </TableRow>
    <TableRow>
      <TableCell>YT burned</TableCell>
      <TableCell><Formula>{"105 \\text{ YT}"}</Formula></TableCell>
    </TableRow>
    <TableRow>
      <TableCell>Current PY Index</TableCell>
      <TableCell><Formula>{"1.08"}</Formula></TableCell>
    </TableRow>
    <TableRow>
      <TableCell>SY received</TableCell>
      <TableCell><Formula>{"\\frac{105}{1.08} = \\textbf{97.2 SY}"}</Formula></TableCell>
    </TableRow>
  </TableBody>
</Table>

<Callout type="warning">
Note: You may receive slightly less SY than you deposited if the PY Index has increased. This is because some yield was distributed to YT holders.
</Callout>

## Post-Expiry Redemption

After maturity, **PT only** can be redeemed:

<Formula display>{"\\text{Underlying}_{received} = \\text{PT}_{amount} \\times 1.0"}</Formula>

### Key Points

- **1:1 redemption** — Each PT redeems for 1 unit of underlying
- **No YT needed** — YT expires worthless
- **Direct to underlying** — Receive the base asset, not SY

## YT Yield Collection

YT holders can claim accrued yield at any time:

<Formula display>{"\\text{Yield}_{claimable} = \\text{YT}_{balance} \\times \\frac{\\text{Index}_{current} - \\text{Index}_{claimed}}{\\text{Index}_{current}}"}</Formula>

Yield is paid in the **underlying asset**.
